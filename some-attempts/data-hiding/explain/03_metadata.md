# 03 — 元数据载体（Metadata as a Carrier）

对应样本：`attack/03_metadata_carrier_SIMULATED.json`

## 这类“攻击”在做什么
攻击者把信息塞进“看起来不影响业务”的字段里，例如：
- 文档/图片/PDF 的元数据字段
- 自定义 header（`X-*`）
- JSON 里的“扩展字段”（`x-...`、`debug`、`comment`）

很多系统会保留并转发这些字段，导致隐藏信息跨系统传播。

> 本样本只使用占位值（placeholder），不包含真实编码后的秘密。

## 典型风险场景
- **跨系统携带**：A 系统写入元数据，B 系统原样存储，C 系统导出时带出。
- **审计盲区**：业务字段被重点审计，但扩展字段未纳入。

## 可观测信号
- 出现与业务无关的 `x-*` 字段或 debug 字段。
- 字段值呈现“高熵/随机样式”（长串字符、异常编码）。
- 同类文档在元数据字段上差异很大。

## 防护与缓解
- **元数据清理**：在上传/导出链路统一做 metadata stripping/normalization。
- **字段白名单**：只允许预期字段通过；丢弃未知字段。
- **日志策略**：把扩展字段纳入审计日志（注意隐私与合规）。

## 练习建议
- 针对 `suspicious_metadata` 这类结构做规则检测并输出告警理由。
- 设计“允许字段列表”与“阻断策略”的分级处置流程。
