# 01 — 零宽字符隐写（Zero‑Width / Invisible Unicode）

对应样本：`attack/01_zero_width_SIMULATED.txt`

## 这类“攻击”在做什么（通俗版）
攻击者把“看不见的字符”夹在正常文本中间（人眼几乎无法察觉），把文本当作载体来偷偷携带信息。常见载体是聊天消息、工单、代码评审评论、邮件正文等。

> 本仓库样本是**模拟**：使用 `[ZWSP]`、`[ZWJ]`、`[ZWNJ]` 等可见标记代替真实不可见字符，避免产生可直接滥用的隐蔽通信载荷。

## 典型风险场景
- **数据外传**：将敏感标识、令牌、短指令夹在普通文本中绕过关键字扫描。
- **审计绕过**：对比工具或人工肉眼检查很难发现异常。
- **隐蔽控制**：在看似无害的文本中嵌入“控制位”或触发条件（例如在流程系统里传播）。

## 可观测信号（Detection Ideas）
- 文本中存在异常 Unicode code point（例如零宽空格类字符）。
- 字符串长度异常（视觉字符数量很少但字节数更大）。
- 同一用户/同一通道出现高频不可见字符或重复模式。

## 防护与缓解（Mitigations）
- **规范化/净化**：对外发文本进行 Unicode 规范化，移除零宽字符或替换为占位符。
- **DLP/审计规则**：加入“不可见字符出现次数阈值”告警。
- **输入校验**：对关键系统字段（用户名、工单标题、配置项）做字符集限制。

## 练习建议（使用本样本）
- 编写检测：扫描是否包含 `[ZWSP]` 等标记（训练用）。
- 评估处置：确认告警后是否应阻断、记录、上报。
