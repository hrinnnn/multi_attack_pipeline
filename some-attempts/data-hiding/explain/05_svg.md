# 05 — SVG 作为载体（Vector Image / SVG Carrier）

对应样本：`attack/05_svg_noise_SIMULATED.svg`

## 这类“攻击”在做什么
SVG 是文本格式的矢量图，天然适合作为“可读的图片文件”。攻击者可能把信息隐藏在：
- 冗余的节点/属性（`data-*`、`id`、`class`）
- 超长的 path 数据
- 大量不可见对象（透明/极小尺寸）
- 注释块

这些内容对渲染结果影响很小，但能携带额外文本。

> 本样本只放了占位 `data-*` 字段与注释，不包含真实编码信息。

## 典型风险场景
- **前端资源投毒**：把“携带信息”的 SVG 当作普通 icon 上传到静态资源库。
- **跨系统传播**：SVG 在多个系统间复制/缓存，导致载体广泛扩散。

## 可观测信号
- SVG 文件出现大量 `data-*` 字段、注释，或对象数量异常。
- path 数据长度异常，或包含与业务无关的文本片段。
- SVG 的视觉效果简单，但文件体积明显偏大。

## 防护与缓解
- **SVG 清理/最小化**：上传前对 SVG 进行 sanitizer/optimizer（会移除多余属性与注释）。
- **策略限制**：对允许上传的图像类型做限制，或将 SVG 转换为安全位图格式。
- **审计**：对 SVG 做结构化解析，统计节点数/属性数/注释比例。

## 练习建议
- 编写检测：统计 `data-` 属性数量、注释块数量、path 长度。
- 处置策略：对超出阈值的 SVG 做隔离与人工复核。
